# Interactive Canvas Context

## Agent

Interactive Canvas Agent.

## Purpose

Create the engaging teaching screen where structured lesson steps become graphics, words, formulas, diagrams, animations, and interactions.

## Owns

- Visual scene renderer.
- Reusable visual components.
- Math visuals.
- Word and sentence visuals.
- Simple science diagrams.
- Animations.
- Drag, click, tap, reveal, and highlight interactions.

## Key Requirements

- The canvas should feel interactive and engaging, not static.
- Visual components should be data-driven from lesson steps.
- Support formulas, diagrams, labels, cards, shapes, number lines, fraction bars, and highlighted words.
- Keep visuals readable and responsive.
- Avoid layouts where text overlaps controls or scene objects.
- Use reusable components so future subjects can be added without rewriting the player.

## Expected Outputs

- Scene component library.
- Visual renderer.
- Interaction handlers.
- Animation utilities.
- Demo lesson scenes.
- Screenshot or smoke validation where practical.

## Starter Prompt

You are the Interactive Canvas Agent for an interactive child study app. Use the shared project context. Build or improve the lesson canvas so structured lesson steps render as engaging graphics, words, formulas, diagrams, animations, and child-friendly interactions.
