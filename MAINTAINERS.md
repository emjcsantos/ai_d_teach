# Maintainers

## Primary Maintainer

- `@emjcsantos`

## Maintenance Responsibilities

- Review pull requests for safety, privacy, lesson quality, and maintainability.
- Triage bug reports and roadmap requests.
- Keep dependencies and public repository settings healthy.
- Preserve local-first defaults and server-side API key handling.
- Maintain release notes as the project moves from prototype to usable learning tool.

## Maintainer Workflow

1. Keep `main` releasable.
2. Use short-lived task branches for focused changes.
3. Run `npm.cmd run check` before merging.
4. Prefer small issues and pull requests that can be reviewed independently.
5. Treat security and child privacy reports as high priority.
